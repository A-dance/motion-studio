## 1行コンセプト — v3（現行）

**3D アバターを自分で動かし、カウントごとのポーズとメモを残す「振り付けメモ帳」。**

---

## 人間 vs アプリの役割分担

| 人間がやること | アプリがやること |
|---------------|----------------|
| 振りを決め、アバターの関節をドラッグしてポーズを作る | IK で関節チェーンを自動解決・3D 表示 |
| フレーズ名・カウントを割り当てる | フレーズ／カウントタイムライン管理 |
| メモ・矢印・回転記号をアノテーションする | Canvas に矢印・テキストを重ね描き |
| 動画を横で見て解釈する | 動画は読まない（動画の解釈は人間） |

## IK の仕様

`applyChainIK(pose, draggedJointId, newPos)` — `src/lib/pose.ts`

対応チェーン:
- 肩 → 肘 → 手首 → 手のひら（右腕・左腕）
- 股 → 膝 → 足首 → 足先（右足・左足）

| ドラッグした関節 | 挙動 |
|----------------|------|
| ルート（肩・股） | チェーン全体を平行移動 |
| 中間（肘・膝） | 直接移動、下段も追従 |
| 末端（手首・足首） | 2ボーン IK で肘・膝を自動解決 |
| 終端（手のひら・足先） | IK + 末端を延伸 |

ポールベクター: 現在の肘・膝位置から自動導出（肘が不自然に裏返らない）。

## 開発時の判断基準

新機能を足す前に:

1. 譜面 JSON のパイプラインを壊さないか？
2. 「掴んで動かす」直感操作を複雑にしないか？
3. 練習の役に立つか（ダッシュボードや統計ではなく）？

3 つとも No ならキルか後回し。

## 動画自動化の位置づけ

| Path | 入力 | 位置づけ |
|------|------|---------|
| **A（当面）** | アバターを手動でドラッグしてポーズ作成 | **メイン** |
| **B（補助）** | MediaPipe で下書き生成 → 手直し | 補助ツール（任意） |
| **C（凍結）** | 動画から全自動でカウント譜面を生成 | Phase 3 以降・凍結 |

動画自動を**唯一の入口にしない**。精度は約束しない。
